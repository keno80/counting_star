# 日志规范（debug 日志 / release 关闭）

## 目标

- 统一日志级别与格式
- release 版本默认关闭调试日志
- 关键错误可定位与追踪

## 范围

- 允许在 UI 与 data 层输出日志
- domain 层禁止依赖 Android 日志
- core 层仅放纯 Kotlin 通用工具

## 日志级别

- DEBUG：开发调试信息，仅 debug 开启
- INFO：关键流程信息，仅 debug 开启
- WARN：可恢复问题，debug 开启
- ERROR：不可恢复问题，debug 开启

## 开关策略

- debug 构建：允许输出
- release 构建：全部关闭

## 约束

- 禁止打印用户隐私与敏感字段
- 禁止输出完整备份内容或明文账号
- 日志 tag 使用模块或功能名
