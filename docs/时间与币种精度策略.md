# 时间与币种精度策略（金额用“分”整型）

## 目标

- 避免浮点误差
- 统一时间与金额的存储口径
- 便于统计与导出

## 金额策略

- 数据库与 domain 使用“分”为单位的 Int/Long
- UI 展示时再格式化为小数
- 计算与汇总禁止使用 Double

## 示例

- 12.34 元存储为 1234
- 加法：1234 + 500 = 1734
- 展示：1734 -> "17.34"

## 货币策略

- MVP 仅支持单币种
- 币种字段仍保留，便于后续扩展

## 时间策略

- 数据库存 UTC 的 epoch millis
- UI 按设备时区显示
- 计算统计时统一以 epoch millis 为基准
